クラスをカテゴライズして管理する。※名前が被ることを防ぐ。アクセス制御も行う。
階層構造はピリオドで区切って表現。
import java.util.Scanner	など。
※実際Timerクラスが、Javaには複数存在するが、パッケージが異なるので、問題がない。

～アクセス制御～
public修飾子を使っていた。
publicを付けずに宣言したクラスには、そのクラスが属するパッケージからは利用できるが、パッケージ外からは不可。

①　import パッケージ名.クラス名;
②　import パッケージ名.*;

②では、パッケージのすべてのクラスをインポートする。※タイプ・インポート・オン・デマンド宣言
※多様は非推奨。▶多用すると、クラス名が　衝突することがある。※どちらのパッケージを使えばいいかわからなくなる。


==パッケージの作成==
パッケージに登録するクラスのソースファイル内で、パッケージを宣言しておく。
フォルダを作成して、パッケージに属するクラスのソースファイルをそこに格納しておく。

package パッケージ名;

クラスへのアクセス制御
クラス定義の時、public を付ける▶パッケージ外からアクセス可能
付けない▶パッケージ内からはアクセス可能
