戻り値を持たないメゾッドは、戻り値のデータ型をvoidとする（メインメゾッドもそう）

public static void ****(~~~~~~) {
~~~~~~~~~
}

javaでは、一つのクラスの中で、同名のメゾッドを複数作成することが許可されている。
メゾッドを多重定義できる。（オーバーロード）
→引数の個数や型を見て、一致するメゾッドを自動的の呼び出してくれる。
（いうなれば、個数と型が完全に一致してしまうとエラーになるということだと。）

戻り値のないメゾッドの場合、return文は書いても書かなくてもいい。
自動的にmainメゾッドに戻る仕組みになっている。


テスト
メゾッドを使用する利点として、処理機能を個々のメゾッドにまとめることが出来ル。
戻り値がない場合は、戻り値を入力する必要はない。
戻り値がない場合は、return文は必須ではない。
